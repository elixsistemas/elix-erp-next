Users — Data Model
Overview

A tabela users representa as contas de usuários do sistema Elix ERP Next.

Usuários são entidades globais do sistema, ou seja, um usuário pode potencialmente acessar múltiplas empresas, dependendo dos vínculos definidos em user_companies.

Esta tabela armazena apenas identidade e autenticação, enquanto permissões e vínculos organizacionais são controlados em outras tabelas.

Table: dbo.users
CREATE TABLE [dbo].[users](
    [id] [int] IDENTITY(1,1) NOT NULL,
     NOT NULL,
     NOT NULL,
     NOT NULL,
    [active] [bit] NOT NULL,
     NOT NULL,
    [default_company_id] [int] NULL
)
Column Reference
Column	Type	Required	Description
id	int	yes	Unique identifier of the user
name	nvarchar(120)	yes	Full name of the user
email	nvarchar(160)	yes	Unique email used for login
password_hash	nvarchar(255)	yes	Secure password hash
active	bit	yes	Indicates if the user account is active
created_at	datetime2	yes	Timestamp of user creation
default_company_id	int	no	Default company selected when the user logs in
Default Constraints
DEFAULT (1) FOR active
DEFAULT (sysutcdatetime()) FOR created_at

Meaning:

New users are active by default

Creation timestamp is automatically generated using UTC

Foreign Keys
FK_users_default_company
users.default_company_id → companies.id

This column defines the default company context for the user.

Important:

This does NOT define the full relationship between users and companies.

The real membership relation should be defined in:

user_companies
Multi-Company Architecture

In Elix ERP Next the relationship works as follows:

users
   │
   ├── user_companies
   │       │
   │       └── companies

Meaning:

A user can belong to multiple companies

Each company can have multiple users

default_company_id defines which company loads first

Example scenario:

User	Companies
João	Contabilidade A, Contabilidade B
Maria	Supermercado X
Carlos	Supermercado X, Supermercado Y
Account Lifecycle

User status is controlled by the active column.

Value	Meaning
1	Active user
0	Disabled user

Inactive users cannot authenticate.

Licensing Integration

The user count per company is used to enforce SaaS licensing limits.

User seats are counted via:

user_companies

Query example:

SELECT COUNT(1)
FROM user_companies
WHERE company_id = @companyId
AND active = 1

If the number exceeds the license limit, new users cannot be activated.

Security Considerations

Passwords are never stored in plain text.

password_hash must contain hashes generated by a secure algorithm such as:

bcrypt

argon2

scrypt

Recommended:

argon2id
Index Recommendations

For performance:

CREATE UNIQUE INDEX UX_users_email
ON dbo.users(email);

Ensures that each email address is unique.

Architectural Notes

The separation between users, roles, permissions, and company membership allows the ERP to support:

multi-tenant SaaS architecture

RBAC (role-based access control)

feature gating by company

licensing per organization
